AI는 30초 만에 랜딩 페이지를 생성할 수 있습니다. 문제는? 다른 모든 사람도 할 수 있다는 것. 그리고 전부 똑같아 보이기 시작합니다: 그라디언트 히어로 섹션, 3열 기능 그리드, "가장 인기" 배지가 있는 가격표, 추천 캐러셀.

코드는 작동합니다. 레이아웃은 합리적입니다. 하지만 랜딩 페이지가 v0, Vercel AI, GPT-4로 만든 다른 10,000개처럼 보입니다. 잠재 고객은 당신을 구별할 수 없습니다.

이건 AI의 잘못이 아닙니다. 제약 문제입니다. 디자인 경계 없이 AI는 랜딩 페이지가 어떻게 보이는지의 통계적 평균을 생성합니다. 이 가이드는 독특하고 브랜드에 맞으며 명백히 AI 생성으로 보이지 않는 랜딩 페이지를 만들기 위해 AI 생성을 제약하는 방법을 보여줍니다.

## 일반적인 AI 랜딩 페이지 문제

v0나 Claude에게 "SaaS 랜딩 페이지 만들어줘"라고 하면 이걸 얻습니다:

```tsx
<section className="bg-gradient-to-r from-blue-600 to-purple-600 text-white py-20">
  <div className="max-w-6xl mx-auto px-6 text-center">
    <h1 className="text-5xl font-bold mb-6">
      [당신의 산업]의 미래
    </h1>
    <p className="text-xl mb-8 max-w-2xl mx-auto">
      [일]을 하는 방식을 혁신하는 혁명적 플랫폼
    </p>
    <button className="bg-white text-blue-600 px-8 py-4 rounded-full text-lg font-semibold">
      무료 체험 시작
    </button>
  </div>
</section>
```

이어서:

- 아이콘이 있는 3개 기능 박스
- 회색조 소셜 증명 로고
- 번호가 매겨진 단계가 있는 "작동 방식"
- 가격표 (무료, 프로, 엔터프라이즈)
- FAQ 아코디언
- 그라디언트가 있는 CTA 푸터

이 구조는 틀리지 않았습니다. 전환도 괜찮습니다. 하지만 경쟁사 랜딩 페이지가 정확히 같은 레이아웃이면 카피로만 경쟁하는 것입니다. 그들도 카피에 AI를 사용한다면 완전히 상품화됩니다.

## 랜딩 페이지를 독특하게 만드는 것

창의적 실험이 아닙니다—**제약된 일관성**. 최고의 랜딩 페이지는 참신함으로 놀라게 하지 않습니다. 일관되고 의도적이며 브랜드와 일치한다고 느껴집니다.

### 1. 시각적 정체성

랜딩 페이지는 제품의 개성을 반영해야 합니다:
- **B2B SaaS:** 깔끔, 전문적, 신뢰 중심
- **개발자 도구:** 코드 중심, 기술적, 군더더기 없음
- **크리에이티브 도구:** 표현적, 유쾌, 비주얼 우선
- **금융:** 안전, 안정, 데이터 중심

AI는 기본적으로 당신의 개성을 모릅니다. 알려줘야 합니다.

### 2. 고유한 제약

일반 랜딩 페이지는 일반 제약 사용:
- 색상: "전문적인 파란색" → `#3B82F6`
- 간격: "충분한 패딩" → 임의 값
- 타이포그래피: "읽을 수 있는" → 16px 본문, 36px 헤딩

독특한 랜딩 페이지는 브랜드 특정 제약 사용:
- 색상: 정확한 팔레트
- 간격: 간격 스케일
- 타이포그래피: 타입 시스템
- 레이아웃: 그리드 선호도

### 3. 콘텐츠 계층

AI는 모든 콘텐츠를 동등하게 취급하는 경향이 있습니다. 독특한 페이지는 명확한 우선순위:
- **주요 CTA:** 명백하고 놓칠 수 없음
- **가치 제안:** 명확하고 앞에 배치
- **소셜 증명:** CTA와 경쟁하지 않고 지원
- **기능:** 훑어볼 수 있고 텍스트 벽이 아님

## 전략 1: 디자인 토큰으로 제약

AI에게 디자인 시스템을 미리 제공해서 처음부터 브랜드에 맞게 생성하도록 합니다.

### 설정: 토큰 기반 프롬프트

이것 대신:
```
프로젝트 관리 도구용 랜딩 페이지 만들어줘
```

이렇게:
```
다음 제약으로 프로젝트 관리 도구용 랜딩 페이지 만들어줘:

색상:
- 배경: bg-base (#FFFFFF)
- 텍스트: text-primary (#111827), text-secondary (#6B7280)
- 브랜드: primary-solid (#7C3AED), primary-fg (#FFFFFF)
- 액센트: accent-solid (#EC4899)

간격 스케일 (4px 기본):
- 섹션: py-16 (64px)
- 카드: p-6 (24px)
- 요소: gap-4 (16px)

타이포그래피:
- 히어로: text-5xl (48px)
- 섹션 헤더: text-3xl (30px)
- 본문: text-lg (18px)

버튼:
- Primary: bg-primary-solid text-primary-fg px-6 py-3 rounded-md
- Secondary: bg-muted text-primary px-6 py-3 rounded-md

그라디언트 금지, 보라-파랑 기본값 금지, 임의 간격 금지.
시맨틱 클래스 이름 사용 (bg-base, text-primary) Tailwind 기본값 아님.
```

**결과:** AI가 일반 Tailwind가 아니라 시스템을 사용하는 페이지 생성.

### 예시 출력

```tsx
<section className="py-16 bg-base">
  <div className="max-w-6xl mx-auto px-6">
    <div className="grid md:grid-cols-2 gap-12 items-center">
      <div>
        <h1 className="text-5xl font-bold text-primary mb-4">
          혼돈 없는 프로젝트 관리
        </h1>
        <p className="text-lg text-secondary mb-6">
          간단한 작업 추적, 명확한 우선순위, 불필요한 일 제로로 
          팀을 정렬 상태로 유지하세요.
        </p>
        <div className="flex gap-4">
          <button className="bg-primary-solid text-primary-fg px-6 py-3 rounded-md font-semibold">
            무료 체험 시작
          </button>
          <button className="bg-muted text-primary px-6 py-3 rounded-md font-semibold">
            데모 보기
          </button>
        </div>
        <p className="text-sm text-tertiary mt-4">
          신용카드 불필요 • 14일 체험
        </p>
      </div>
      <div className="bg-subtle border border-default rounded-lg p-8">
        {/* 제품 스크린샷 또는 일러스트 */}
      </div>
    </div>
  </div>
</section>
```

주목할 점:
- 토큰 기반 클래스 사용 (`bg-base`, `text-primary`)
- 간격 스케일 준수 (`py-16`, `gap-12`, `mb-4`)
- 타이포그래피 스케일 일치 (`text-5xl`, `text-lg`)
- 주요 CTA가 브랜드 색상으로 돋보임
- 그라디언트 없음, 일반 파란색 없음

## 전략 2: 기존 브랜드 참조

AI에게 특정 디자인 언어와 일치하라고 알립니다.

**프롬프트:**
```
Linear의 비주얼 스타일로 랜딩 페이지 만들어줘:
- 미니멀, 여백 많음
- 뮤트된 보라 액센트 (#5E6AD2)
- 깔끔한 산세리프 타이포그래피
- 서브틀한 보더, 그림자 없음
- 주요 비주얼로 제품 스크린샷
- 단순 그리드 레이아웃, 어지럽지 않음
```

또는:

```
Stripe에서 영감받은 랜딩 페이지 만들어줘:
- 전문적, 신뢰 중심
- 보라 브랜드 색상 (#635BFF)
- 명확한 계층
- 비주얼 요소로 코드 스니펫
- 인라인 데모
- 기술적이지만 접근 가능한 톤
```

AI 모델은 잘 알려진 브랜드에 강한 사전 지식을 갖고 있습니다. 참조하면 특정 미학을 향해 생성을 고정합니다.

**경고:** 복사하지 마세요. 복제 대상이 아니라 스타일 참조로 사용.

## 전략 3: 안티 패턴

AI에게 하지 말아야 할 것을 알립니다. 일반 기본값을 제거합니다.

**프롬프트:**
```
개발자 도구용 랜딩 페이지 만들어줘.

사용 금지:
- 그라디언트 배경 (특히 파랑-보라)
- 회의 중인 사람들의 스톡 사진
- 일반 아이콘 (로켓, 번개, 차트 상승)
- "혁명적" 또는 "변화" 언어
- rounded-full 버튼
- 캐러셀 추천

사용:
- 비주얼 요소로 코드 예제
- 적절한 곳에 터미널/CLI 미학
- 기술적 세부사항에 모노스페이스 폰트
- 실제 제품 스크린샷
- 개발자 중심 카피 (과장 아닌 구체적)
- 직사각형, 미니멀 버튼 스타일
```

이렇게 하면 AI가 "일반적인 것"에서 "이 청중에게 독특한 것"으로 바뀝니다.

## 전략 4: FramingUI로 컴포넌트별

전체 페이지를 생성하는 대신 토큰 제약으로 컴포넌트별로 구축합니다.

### 히어로 섹션

**프롬프트:**
```
@design-tokens.md 히어로 섹션 만들어줘:
- 분할 레이아웃: 왼쪽 텍스트, 오른쪽 비주얼
- 큰 헤딩 (text-5xl), 간결한 부제 (text-lg)
- 2개 CTA: primary (bg-primary-solid), secondary (bg-muted)
- CTA 아래 신뢰 지표 (가격, 체험 정보)
- 배경: bg-base, 그라디언트 없음
```

### 기능 그리드

**프롬프트:**
```
@design-tokens.md 기능 섹션 만들어줘:
- 데스크톱 3열 그리드, 모바일 스택
- 각 기능: 아이콘, 헤딩 (text-xl), 설명 (text-base)
- 아이콘: 24px, text-secondary 색상
- 카드: bg-subtle, border-default, rounded-lg, p-6
- 섹션 헤더: text-3xl, text-primary, mb-12
```

### 소셜 증명

**프롬프트:**
```
@design-tokens.md 소셜 증명 섹션 만들어줘:
- 로고 클라우드: 회색조, 동일 크기
- 짧은 추천 인용: text-base, text-secondary
- 고객 아바타: rounded-full, 48px
- 레이아웃: 가운데, max-w-4xl
- 배경: bg-subtle
```

컴포넌트를 별도로 생성하면 모든 섹션에 걸쳐 토큰 일관성을 유지합니다. 모든 것이 같은 시스템을 사용하기 때문에 전체 페이지가 일관되게 느껴집니다.

## 전략 5: 제약으로 반복

첫 생성은 거의 완벽하지 않습니다. 제약으로 개선합니다.

**초기:**
```
이메일 마케팅 도구용 랜딩 페이지 히어로 만들어줘
```

**AI가 일반 그라디언트 히어로 생성**

**개선:**
```
그라디언트 배경 제거. bg-base 사용.
CTA 버튼 더 두드러지게: 더 큰 패딩 (px-8 py-4), 
bg-primary-solid로 더 강한 대비.
더 나은 계층을 위해 헤딩 크기 text-6xl에서 text-5xl로 축소.
CTA 아래 신뢰 지표 추가: "10,000명 이상 마케터 합류"
```

**AI가 제약으로 업데이트**

각 반복이 구체성을 추가합니다. 3-4라운드까지 페이지는 생성된 게 아니라 의도적으로 보입니다.

```tsx
        </p>
        <div className="flex gap-4 items-center mb-4">

            문서 보기

          {`import { tokens } from '@framingui/tokens'

const button = {
  background: tokens.color.primary.solid,
  color: tokens.color.primary.fg,

```

개선:
- 토큰 기반 색상 시스템 (bg-base, text-primary)
- 구체적 가치 제안 ("비즈니스 변화" 아님)
- 깔끔한 미니멀 레이아웃 (그라디언트 없음)
- 개발자 중심 (코드 예제 포함)
- 일관된 간격 (간격 스케일)
- 직사각형 버튼 (rounded-md)
- 생성된 게 아니라 의도적으로 보임

## 도움이 되는 도구

### FramingUI + AI

FramingUI 토큰을 사용해서 AI 생성 제약:

1. **토큰 설치:**
   ```bash
   npm install @framingui/tokens @framingui/tailwind-preset
   ```

2. **`.cursorrules` 또는 `.clauderules` 생성:**
   ```
   랜딩 페이지 섹션 생성 시:
   - 모든 색상, 간격, 타이포그래피에 @framingui/tokens 사용
   - 시맨틱 토큰 참조 (bg-base, text-primary, primary-solid)
   - 간격 스케일 준수 (p-6, gap-4, py-16)
   - 그라디언트 금지, 임의 hex 코드 금지
   ```

3. **컨텍스트로 생성:**
   ```
   @design-tokens.md 랜딩 페이지 히어로 섹션 만들어줘
   ```

AI가 이제 디자인 시스템과 자동으로 일치하는 토큰 기반 코드를 생성합니다.

### 토큰 프리셋이 있는 v0

v0.dev 사용 시:

1. v0 프로젝트 프리셋으로 토큰 정의
2. 모든 생성에서 프리셋 참조
3. v0가 섹션 전체에 일관성 유지

### Figma → 토큰 → AI

Figma 디자인이 있다면:

1. Figma에서 디자인 토큰 내보내기 (Tokens Studio 같은 플러그인으로)
2. FramingUI 호환 형식으로 변환
3. 생성을 위해 AI에 토큰 제공
4. 생성된 코드가 Figma 디자인과 일치

## 일반적인 실수

### 1. 첫 출력 신뢰

AI의 첫 생성은 최종 버전이 아니라 시작점입니다. 항상 제약으로 반복하세요.

### 2. 비주얼 리뷰 없음

생성된 코드는 의미적으로 올바를 수 있지만 시각적으로 벗어날 수 있습니다. 항상 미리보고 조정하세요.

### 3. 일관되지 않은 프롬프트

히어로를 한 프롬프트 스타일로 생성하고 기능을 다른 스타일로 생성하면 섹션이 일치하지 않습니다. 모든 프롬프트에서 일관된 토큰 참조 사용.

### 4. 반응형 디자인 무시

AI는 종종 데스크톱 우선 생성. 명시적으로 모바일 프롬프트:
```
모바일에서 작동하도록 확인: 열 스택, 텍스트 크기 축소, 
작은 화면용 간격 조정
```

## 결과

토큰 제약 AI 생성으로:

✅ **브랜드 일관성** - 모든 섹션이 디자인 시스템 사용
✅ **빠른 반복** - 처음부터 만드는 대신 생성 + 개선
✅ **고유한 미학** - 토큰 = 모양, 일반 기본값 아님
✅ **확장 가능** - 랜딩 페이지 토큰이 전체 제품으로 확장
✅ **유지 관리 가능** - 토큰 업데이트하면 랜딩 페이지 자동 업데이트

## AI 생성 랜딩 페이지 체크리스트

배포 전:

- [ ] 모든 색상이 시맨틱 토큰 사용 (하드코딩된 hex 없음)
- [ ] 간격이 스케일 준수 (임의 값 없음)
- [ ] 타이포그래피가 타입 시스템 사용
- [ ] 주요 CTA가 시각적으로 지배적
- [ ] 카피가 구체적, 유행어 아님
- [ ] 레이아웃이 v0 기본값처럼 보이지 않음
- [ ] 모바일 반응형 (실제 기기에서 테스트)
- [ ] 빠른 로드 (거대한 이미지 없음, 최소 JS)
