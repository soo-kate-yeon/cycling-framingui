Cursor AI는 코드를 빠르게 생성합니다. 하지만 디자인 토큰 없이는 임의의 색상, 간격, 스타일을 사용해서 디자인 시스템과 맞지 않는 코드를 만들어냅니다. 해결책은 매번 Figma 스펙을 프롬프트에 붙여넣는 게 아니라, Cursor가 자동으로 참조할 수 있는 구조화된 토큰 데이터를 제공하는 것입니다.

이 가이드는 FramingUI 디자인 토큰을 Cursor와 연동해서 생성되는 모든 컴포넌트가 실제 디자인 시스템 값을 사용하도록 설정하는 방법을 보여줍니다.

## Cursor에 디자인 토큰이 필요한 이유

Cursor의 AI 모델은 수백만 개의 코드베이스로 학습되었습니다. "버튼 컴포넌트 만들어줘"라고 요청하면, 학습 데이터에서 본 전형적인 버튼 모양을 생성합니다. 즉:

- **일반적인 색상:** 브랜드 블루 대신 `#3B82F6`
- **임의의 간격:** 간격 스케일 대신 `padding: 12px 24px`
- **제멋대로인 radius:** 표준 코너 반경 대신 `border-radius: 6px`
- **불일치하는 네이밍:** 파일마다 `primaryButton`, `main-btn`, `ButtonPrimary` 혼재

디자인 토큰이 컨텍스트에 있으면 Cursor는 이런 컴포넌트를 생성합니다:
- 정확한 컬러 팔레트 사용
- 간격 시스템 준수
- 타이포그래피 스케일 일치
- 시맨틱 토큰 이름 일관되게 참조

## 1단계: FramingUI 토큰 설치

먼저 프로젝트에 FramingUI를 추가합니다:

```bash
npm install @framingui/tokens
# 또는
pnpm add @framingui/tokens
```

Tailwind 프로젝트는:

```bash
npm install @framingui/tailwind-preset
```

## 2단계: 토큰 레퍼런스 파일 생성

프로젝트 루트에 `.cursor/tokens.md`를 만듭니다:

```markdown
# 디자인 시스템 토큰

## 색상

시맨틱 컬러 토큰:

- `color.bg.base` - 메인 배경 (#FFFFFF)
- `color.bg.subtle` - 서브틀한 배경 (#F9FAFB)
- `color.bg.muted` - 뮤트된 배경 (#F3F4F6)
- `color.text.primary` - 주요 텍스트 (#111827)
- `color.text.secondary` - 보조 텍스트 (#6B7280)
- `color.border.default` - 기본 보더 (#E5E7EB)
- `color.primary.solid` - 주요 액션 색상
- `color.primary.fg` - primary 배경 위 텍스트

## 간격

간격 스케일 (기본 4px):

- `spacing.1` = 4px
- `spacing.2` = 8px
- `spacing.3` = 12px
- `spacing.4` = 16px
- `spacing.5` = 20px
- `spacing.6` = 24px
- `spacing.8` = 32px
- `spacing.10` = 40px
- `spacing.12` = 48px

## 타이포그래피

폰트 크기와 line-height:

- `text.xs` - 12px/16px
- `text.sm` - 14px/20px
- `text.base` - 16px/24px
- `text.lg` - 18px/28px
- `text.xl` - 20px/28px
- `text.2xl` - 24px/32px

## Border Radius

- `radius.sm` = 4px
- `radius.md` = 6px
- `radius.lg` = 8px
- `radius.xl` = 12px
- `radius.2xl` = 16px
- `radius.full` = 9999px
```

## 3단계: Cursor 규칙 설정

프로젝트 루트에 `.cursorrules` 생성 또는 업데이트:

```text
# 디자인 시스템 규칙

## 항상 디자인 토큰 사용

UI 컴포넌트 생성 시:
1. @framingui/tokens에서 토큰 import
2. 시맨틱 토큰 이름 사용, 하드코딩된 값 절대 금지
3. .cursor/tokens.md에서 사용 가능한 토큰 참조
4. 간격 스케일 준수 (4px의 배수)

## 코드 스타일

- 토큰은 CSS 변수로: var(--color-bg-base)
- Tailwind: bg-base, text-primary 같은 preset 클래스 사용
- 컴포넌트 props는 토큰 이름을 받아야 함, raw 값 X
- 컴포넌트 주석에 사용된 토큰 문서화

## 예시

좋음:
```tsx
<button className="bg-primary-solid text-primary-fg px-4 py-2 rounded-md">
  클릭
</button>
```

나쁨:
```tsx
<button className="bg-blue-600 text-white px-4 py-2 rounded-md">
  클릭
</button>
```
```

## 4단계: 토큰 설정 구성

CSS-in-JS 프로젝트는 `src/tokens.ts` 생성:

```typescript
import { tokens } from '@framingui/tokens'

export const colors = tokens.color
export const spacing = tokens.spacing
export const typography = tokens.typography
export const radius = tokens.radius

// CSS 변수로 export
export const cssVars = `
  :root {
    --color-bg-base: ${colors.bg.base};
    --color-bg-subtle: ${colors.bg.subtle};
    --color-text-primary: ${colors.text.primary};
    --color-primary-solid: ${colors.primary.solid};
    --spacing-4: ${spacing[4]};
    /* ... 추가 토큰 */
  }
`
```

Tailwind는 `tailwind.config.js` 업데이트:

```javascript
import { framingUIPreset } from '@framingui/tailwind-preset'

export default {
  presets: [framingUIPreset],
  content: ['./src/**/*.{js,jsx,ts,tsx}'],
  // 커스텀 오버라이드
}
```

## 5단계: Cursor로 테스트

이제 Cursor에서 이런 프롬프트를 시도해보세요:

**프롬프트 1:**
```
디자인 토큰을 사용해서 서브틀한 배경, primary 텍스트, 
적절한 간격을 가진 카드 컴포넌트 만들어줘
```

Cursor가 생성해야 할 코드:

```tsx
export function Card({ children }: { children: React.ReactNode }) {
  return (
    <div className="bg-subtle border border-default rounded-lg p-6">
      <div className="text-primary">{children}</div>
    </div>
  )
}
```

**프롬프트 2:**
```
컬러 토큰 사용해서 primary 버튼 컴포넌트 만들어줘
```

예상 출력:

```tsx
export function Button({ children, onClick }: ButtonProps) {
  return (
    <button
      onClick={onClick}
      className="bg-primary-solid text-primary-fg px-4 py-2 rounded-md 
                 hover:opacity-90 transition-opacity"
    >
      {children}
    </button>
  )
}
```

## 6단계: Cursor 토큰 자동완성 활용

토큰을 파일에 import하면 Cursor의 자동완성이 토큰 이름을 제안합니다:

```tsx
// "color." 타이핑하면 Cursor가 제안:
// color.bg.base
// color.bg.subtle
// color.text.primary
// 등등

const cardStyles = {
  backgroundColor: tokens.color.bg.subtle,
  padding: tokens.spacing[4],
  borderRadius: tokens.radius.lg,
}
```

## 고급: Composer에서 토큰 컨텍스트

복잡한 컴포넌트는 Cursor Composer에 토큰 컨텍스트 포함:

```
@tokens.md 다음 조건의 데이터 테이블 만들어줘:
- 헤더는 subtle 배경
- 행 사이 border.default로 구분
- 적절한 간격 (셀은 spacing.4, compact 모드는 spacing.2)
- hover 상태는 bg.muted 사용
```

`@tokens.md` 참조는 Composer에게 생성 전에 토큰 문서를 로드하라고 알려줍니다.

## 일반적인 패턴

### 대시보드 카드

```tsx
// Cursor 프롬프트: "아이콘, 레이블, 값이 있는 통계 카드 만들어줘"
export function StatCard({ icon, label, value }: StatCardProps) {
  return (
    <div className="bg-base border border-default rounded-lg p-6">
      <div className="flex items-center gap-3 mb-2">
        <div className="text-secondary">{icon}</div>
        <span className="text-sm text-secondary">{label}</span>
      </div>
      <div className="text-2xl font-semibold text-primary">{value}</div>
    </div>
  )
}
```

### 폼 입력

```tsx
// Cursor 프롬프트: "레이블과 에러 상태가 있는 텍스트 입력 만들어줘"
export function TextField({ label, error, ...props }: TextFieldProps) {
  return (
    <div className="space-y-2">
      <label className="text-sm font-medium text-primary">{label}</label>
      <input
        {...props}
        className={cn(
          "w-full px-3 py-2 rounded-md border",
          "bg-base text-primary",
          "focus:outline-none focus:ring-2 focus:ring-primary-solid",
          error ? "border-error" : "border-default"
        )}
      />
      {error && <p className="text-sm text-error">{error}</p>}
    </div>
  )
}
```

### 모달 다이얼로그

```tsx
// Cursor 프롬프트: "오버레이, 닫기 버튼, 헤더, 액션이 있는 모달 만들어줘"
export function Modal({ isOpen, onClose, title, children }: ModalProps) {
  if (!isOpen) return null
  
  return (
    <div className="fixed inset-0 z-50 flex items-center justify-center">
      <div className="absolute inset-0 bg-black/50" onClick={onClose} />
      <div className="relative bg-base rounded-xl shadow-xl max-w-md w-full mx-4 p-6">
        <h2 className="text-xl font-semibold text-primary mb-4">{title}</h2>
        <div className="text-secondary mb-6">{children}</div>
        <div className="flex gap-3 justify-end">
          <button className="px-4 py-2 rounded-md bg-muted text-primary">
            취소
          </button>
          <button className="px-4 py-2 rounded-md bg-primary-solid text-primary-fg">
            확인
          </button>
        </div>
      </div>
    </div>
  )
}
```

## 트러블슈팅

### Cursor가 여전히 하드코딩된 값 사용

**체크:**
1. `.cursorrules`가 프로젝트 루트에 있나요?
2. 토큰 파일 추가 후 Cursor 재시작했나요?
3. 프롬프트에 명시적으로 토큰 언급: "@framingui/tokens의 디자인 토큰 사용해서"

### 토큰 이름이 자동완성 안 됨

**해결:**
1. 활성 파일에 토큰이 import되어 있는지 확인
2. 전체 경로 먼저 타이핑: `tokens.color.bg.base`
3. Cursor는 사용에서 배움—토큰을 몇 번 사용하면 자동완성 개선됨

### 생성된 코드가 토큰 스타일 혼용

**솔루션:**
`.cursorrules`를 더 명시적으로 업데이트:

```text
절대 사용 금지:
- 하드코딩된 hex 색상 (#FFFFFF, #000000 등)
- 임의의 픽셀 값 (12px, 18px 등)
- 일반 Tailwind 색상 (bg-blue-500, text-gray-600)

항상 사용:
- 토큰 클래스 (bg-base, text-primary)
- 간격 스케일 (p-4, gap-6, space-y-3)
- 디자인 시스템의 시맨틱 이름
```

## 얻을 수 있는 것

FramingUI 토큰 + Cursor 조합:

✅ **디자인 드리프트 제로** - 생성된 모든 컴포넌트가 시스템 사용
✅ **빠른 반복** - Cursor가 즉시 프로덕션 수준 코드 생성
✅ **일관된 네이밍** - AI 생성 파일 전체에 시맨틱 토큰
✅ **쉬운 리팩토링** - 토큰 한 번 변경으로 전체 업데이트
✅ **더 나은 프롬프트** - 비주얼 스펙 대신 기능에 집중
