# Spacing System and Layout Tokens

FramingUI uses a predictable spacing system based on an **8px grid** — providing consistent rhythm and making design decisions easier for both humans and AI.

## Why an 8px Grid?

The 8px grid system is an industry standard because:

- ✅ **Scalable** — Divides evenly at common screen sizes
- ✅ **Accessible** — Large enough touch targets (44px minimum = 5.5 units)
- ✅ **Simple math** — Easy mental calculations (2×, 3×, 4×, etc.)
- ✅ **Flexible** — Works for spacing, sizing, and alignment

**Core principle:** All spacing and sizing values should be multiples of 8px (or 4px for fine adjustments).

## Spacing Tokens

FramingUI provides spacing tokens from **4px to 96px**:

| Token | Rem | Pixels | Usage |
| --- | --- | --- | --- |
| `--spacing-1` | 0.25rem | 4px | Tight spacing, icon gaps |
| `--spacing-2` | 0.5rem | 8px | Compact layouts, badges |
| `--spacing-3` | 0.75rem | 12px | Small buttons, inline elements |
| `--spacing-4` | 1rem | 16px | Default button padding |
| `--spacing-5` | 1.25rem | 20px | Comfortable padding |
| `--spacing-6` | 1.5rem | 24px | Card padding, sections |
| `--spacing-8` | 2rem | 32px | Large gaps, page margins |
| `--spacing-10` | 2.5rem | 40px | Section spacing |
| `--spacing-12` | 3rem | 48px | Major section breaks |
| `--spacing-16` | 4rem | 64px | Hero section padding |
| `--spacing-20` | 5rem | 80px | Large whitespace |
| `--spacing-24` | 6rem | 96px | Extra-large spacing |

## Using Spacing Tokens

### In CSS

```css
.card {
  padding: var(--spacing-6);          /* 24px all sides */
  margin-bottom: var(--spacing-8);    /* 32px below */
  gap: var(--spacing-4);              /* 16px between children */
}

.button {
  padding: var(--spacing-3) var(--spacing-6);  /* 12px 24px */
  gap: var(--spacing-2);              /* 8px icon-text gap */
}

.section {
  padding-block: var(--spacing-16);   /* 64px top/bottom */
  padding-inline: var(--spacing-6);   /* 24px left/right */
}
```

### In Tailwind (via FramingUI)

FramingUI maps spacing tokens to Tailwind utilities:

```tsx
<div className="p-6 gap-4">
  {/* p-6 = padding: var(--spacing-6) → 24px */}
  {/* gap-4 = gap: var(--spacing-4) → 16px */}
  
  <h2 className="mb-4">Heading</h2>
  {/* mb-4 = margin-bottom: 16px */}
  
  <p className="mb-8">Paragraph</p>
  {/* mb-8 = margin-bottom: 32px */}
</div>
```

### In Components

```tsx
import { Button, Card } from '@framingui/ui';

export function PricingCard() {
  return (
    <Card className="p-6 space-y-4">
      {/* p-6 = 24px padding, space-y-4 = 16px vertical gaps */}
      
      <h3 className="text-2xl font-bold">Pro Plan</h3>
      <p className="text-muted-foreground">$29/month</p>
      
      <ul className="space-y-2">
        {/* space-y-2 = 8px gaps between items */}
        <li>Unlimited projects</li>
        <li>Priority support</li>
      </ul>
      
      <Button className="mt-6">Subscribe</Button>
      {/* mt-6 = 24px top margin */}
    </Card>
  );
}
```

## Spacing Scale Guidelines

### Micro Spacing (1-3)

**4px, 8px, 12px** — Use for:
- Icon-text gaps
- Badge padding
- Inline element spacing
- Form field internal spacing

```tsx
<Button className="gap-2">
  {/* 8px between icon and text */}
  <Icon />
  <span>Label</span>
</Button>
```

### Component Spacing (4-6)

**16px, 20px, 24px** — Use for:
- Button padding
- Card padding
- Input padding
- List item gaps

```tsx
<Card className="p-6">
  {/* 24px padding */}
  <div className="space-y-4">
    {/* 16px gaps between elements */}
    <h3>Title</h3>
    <p>Content</p>
  </div>
</Card>
```

### Layout Spacing (8-12)

**32px, 40px, 48px** — Use for:
- Section margins
- Grid gaps
- Column spacing
- Major content breaks

```tsx
<section className="py-12 px-6">
  {/* 48px vertical, 24px horizontal */}
  <div className="grid grid-cols-3 gap-8">
    {/* 32px between grid items */}
    <Card>...</Card>
    <Card>...</Card>
    <Card>...</Card>
  </div>
</section>
```

### Macro Spacing (16-24)

**64px, 80px, 96px** — Use for:
- Hero section padding
- Page section separation
- Large breakout areas
- Footer spacing

```tsx
<section className="py-20">
  {/* 80px vertical padding */}
  <div className="max-w-6xl mx-auto">
    <h1>Hero Title</h1>
  </div>
</section>
```

## Layout Patterns

### Stack (Vertical Spacing)

Use `space-y-*` for vertical rhythm:

```tsx
<div className="space-y-4">
  <h2>Heading</h2>
  <p>Paragraph 1</p>
  <p>Paragraph 2</p>
  <Button>Action</Button>
</div>
```

Equivalent CSS:

```css
.stack > * + * {
  margin-top: var(--spacing-4);
}
```

### Cluster (Horizontal Spacing)

Use `space-x-*` or `gap-*` for horizontal rhythm:

```tsx
<div className="flex gap-4">
  <Button>Cancel</Button>
  <Button>Save</Button>
  <Button>Submit</Button>
</div>
```

### Grid

Use `gap-*` for consistent spacing:

```tsx
<div className="grid grid-cols-3 gap-6">
  <Card>Item 1</Card>
  <Card>Item 2</Card>
  <Card>Item 3</Card>
</div>
```

### Container Padding

Use `p-*` or `px-*`, `py-*` for container padding:

```tsx
<section className="px-6 py-12">
  {/* 24px horizontal, 48px vertical */}
  <div className="max-w-6xl mx-auto">
    Content
  </div>
</section>
```

## Responsive Spacing

Adjust spacing at different breakpoints:

```tsx
<section className="px-4 md:px-6 lg:px-8 py-8 md:py-12 lg:py-16">
  {/* Mobile: 16px horizontal, 32px vertical */}
  {/* Tablet: 24px horizontal, 48px vertical */}
  {/* Desktop: 32px horizontal, 64px vertical */}
  
  <h1 className="mb-4 md:mb-6 lg:mb-8">
    {/* Mobile: 16px, Tablet: 24px, Desktop: 32px bottom margin */}
    Responsive Heading
  </h1>
</section>
```

## Custom Spacing

### Add Custom Spacing Tokens

```css
/* app/globals.css */
:root {
  --spacing-7: 1.75rem;   /* 28px - between 6 and 8 */
  --spacing-9: 2.25rem;   /* 36px - between 8 and 10 */
  --spacing-15: 3.75rem;  /* 60px - between 12 and 16 */
}
```

Use in Tailwind config:

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      spacing: {
        '7': 'var(--spacing-7)',
        '9': 'var(--spacing-9)',
        '15': 'var(--spacing-15)',
      },
    },
  },
};
```

## Touch Target Guidelines

**Minimum touch target:** 44px × 44px (WCAG AAA)

```tsx
{/* Button with minimum touch area */}
<Button className="min-h-[44px] px-6">
  Tap Me
</Button>

{/* Icon button with adequate touch area */}
<button className="w-11 h-11 flex items-center justify-center">
  {/* 44px × 44px */}
  <Icon size={20} />
</button>
```

## Common Spacing Anti-Patterns

### ❌ Avoid Magic Numbers

```css
/* Bad - arbitrary values */
.card {
  padding: 23px;
  margin-bottom: 17px;
}
```

```css
/* Good - token-based */
.card {
  padding: var(--spacing-6);      /* 24px */
  margin-bottom: var(--spacing-4); /* 16px */
}
```

### ❌ Avoid Inconsistent Gaps

```tsx
{/* Bad - inconsistent spacing */}
<div className="space-y-3">
  <p className="mb-5">Text 1</p>
  <p className="mb-2">Text 2</p>
  <p className="mb-7">Text 3</p>
</div>
```

```tsx
{/* Good - consistent rhythm */}
<div className="space-y-4">
  <p>Text 1</p>
  <p>Text 2</p>
  <p>Text 3</p>
</div>
```

### ❌ Avoid Asymmetric Padding

```tsx
{/* Bad - unbalanced */}
<Card className="pt-8 pb-4 pl-6 pr-3">
  Content
</Card>
```

```tsx
{/* Good - balanced */}
<Card className="p-6">
  Content
</Card>
```

## Layout Token Best Practices

1. **Use the scale** — Stick to predefined tokens
2. **Start with defaults** — `p-6`, `gap-4`, `space-y-4` cover 80% of cases
3. **Add spacing, don't subtract** — Prefer margins over negative values
4. **Test on mobile** — Verify touch targets and spacing on small screens
5. **Maintain rhythm** — Consistent spacing creates visual harmony
