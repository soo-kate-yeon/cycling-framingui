# Typography Scale and Font Tokens

FramingUI provides a carefully crafted typography system with fluid scaling, semantic font tokens, and accessible type hierarchy.

## Typography Tokens Overview

FramingUI uses tokens for fonts, sizes, weights, and line heights:

```css
/* Font families */
--font-sans: 'Inter', system-ui, sans-serif;
--font-mono: 'Fira Code', 'Courier New', monospace;

/* Font sizes */
--text-xs: 0.75rem;    /* 12px */
--text-sm: 0.875rem;   /* 14px */
--text-base: 1rem;     /* 16px */
--text-lg: 1.125rem;   /* 18px */
--text-xl: 1.25rem;    /* 20px */
--text-2xl: 1.5rem;    /* 24px */
--text-3xl: 1.875rem;  /* 30px */
--text-4xl: 2.25rem;   /* 36px */

/* Font weights */
--font-normal: 400;
--font-medium: 500;
--font-semibold: 600;
--font-bold: 700;

/* Line heights */
--leading-none: 1;
--leading-tight: 1.25;
--leading-snug: 1.375;
--leading-normal: 1.5;
--leading-relaxed: 1.75;
```

## Type Scale Ratios

FramingUI uses a **1.25x scale ratio** (major third) for harmonious size relationships:

| Token | Size | Ratio | Usage |
| --- | --- | --- | --- |
| `--text-xs` | 12px | 0.75× | Captions, labels |
| `--text-sm` | 14px | 0.875× | Body small, UI text |
| `--text-base` | 16px | 1× | Body text |
| `--text-lg` | 18px | 1.125× | Lead paragraphs |
| `--text-xl` | 20px | 1.25× | Subheadings |
| `--text-2xl` | 24px | 1.5× | H3 headings |
| `--text-3xl` | 30px | 1.875× | H2 headings |
| `--text-4xl` | 36px | 2.25× | H1 headings |

**Why 1.25x?** It creates visual rhythm without excessive jumps between sizes.

## Fluid Typography

FramingUI supports **fluid type scaling** — text sizes adapt between minimum and maximum viewport widths:

```css
/* Basic fluid type */
:root {
  --text-base: clamp(0.9375rem, 0.875rem + 0.3125vw, 1rem);
  /* Scales from 15px (mobile) to 16px (desktop) */
  
  --text-2xl: clamp(1.375rem, 1.25rem + 0.625vw, 1.5rem);
  /* Scales from 22px (mobile) to 24px (desktop) */
  
  --text-4xl: clamp(2rem, 1.75rem + 1.25vw, 2.25rem);
  /* Scales from 32px (mobile) to 36px (desktop) */
}
```

**How `clamp()` works:**

```css
clamp(MIN, PREFERRED, MAX)
```

- `MIN` — Smallest size (mobile)
- `PREFERRED` — Viewport-based calculation
- `MAX` — Largest size (desktop)

### Enable Fluid Typography

Override tokens in your global CSS:

```css
/* app/globals.css */
:root {
  /* Fluid body text: 15px → 16px */
  --text-base: clamp(0.9375rem, 0.875rem + 0.3125vw, 1rem);
  
  /* Fluid headings */
  --text-xl: clamp(1.125rem, 1rem + 0.625vw, 1.25rem);
  --text-2xl: clamp(1.375rem, 1.25rem + 0.625vw, 1.5rem);
  --text-3xl: clamp(1.75rem, 1.5rem + 1.25vw, 1.875rem);
  --text-4xl: clamp(2rem, 1.75rem + 1.25vw, 2.25rem);
}
```

```css

  font-weight: var(--font-bold);
  line-height: var(--leading-tight);

  font-size: var(--text-3xl);

  line-height: var(--leading-snug);
```

## Font Weight Usage

| Token | Value | Usage |
| --- | --- | --- |
| `--font-normal` | 400 | Body text, paragraphs |
| `--font-medium` | 500 | Emphasized text, labels |
| `--font-semibold` | 600 | Subheadings, buttons |
| `--font-bold` | 700 | Headings, strong emphasis |

**Best practices:**

- Use `--font-normal` for body text
- Reserve `--font-bold` for headings and strong emphasis
- Use `--font-medium` for UI elements (buttons, tabs, labels)

## Line Height Guidelines

| Token | Value | Usage |
| --- | --- | --- |
| `--leading-none` | 1.0 | Badges, tight layouts |
| `--leading-tight` | 1.25 | Headings, titles |
| `--leading-snug` | 1.375 | Subheadings, UI text |
| `--leading-normal` | 1.5 | Body text (default) |
| `--leading-relaxed` | 1.75 | Long-form content |

**Rule of thumb:**

- Larger text → Tighter line height
- Body text → Normal line height (1.5)
- Dense content → Relaxed line height (1.75)

## Customizing Font Families

### Use a Custom Google Font

```css
/* app/globals.css */
@import url('https://fonts.googleapis.com/css2?family=Poppins:wght@400;500;600;700&display=swap');

:root {
  --font-sans: 'Poppins', system-ui, sans-serif;
}
```

```tsx
// next.config.js (optimize font loading)
module.exports = {
  experimental: {
    optimizeFonts: true,
  },
};
```

### Use Next.js Font Optimization

```tsx
// app/layout.tsx
import { Inter, Fira_Code } from 'next/font/google';

const inter = Inter({ 
  subsets: ['latin'],
  variable: '--font-sans',
});

const firaCode = Fira_Code({
  subsets: ['latin'],
  variable: '--font-mono',
});

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={`${inter.variable} ${firaCode.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

Then reference the CSS variables:

```css
/* app/globals.css */
:root {
  --font-sans: var(--font-sans);  /* Now uses next/font optimized */
  --font-mono: var(--font-mono);
}
```

### Use a Local Font

```css
/* app/globals.css */
@font-face {
  font-family: 'CustomSans';
  src: url('/fonts/CustomSans-Regular.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

@font-face {
  font-family: 'CustomSans';
  src: url('/fonts/CustomSans-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap;
}

:root {
  --font-sans: 'CustomSans', system-ui, sans-serif;
}
```

## Responsive Typography Patterns

### Responsive Headings

```tsx
<h1 className="text-2xl md:text-3xl lg:text-4xl font-bold">
  Responsive Heading
</h1>
```

### Responsive Line Height

```css
.article-body {
  font-size: var(--text-base);
  line-height: var(--leading-normal);
}

@media (min-width: 768px) {
  .article-body {
    line-height: var(--leading-relaxed);
  }
}
```

### Responsive Measure (Line Length)

Control text width for readability:

```tsx
<article className="max-w-prose mx-auto">
  <p className="text-base leading-relaxed">
    Long-form content automatically constrains to ~65 characters per line
    for optimal readability.
  </p>
</article>
```

`max-w-prose` = ~65ch (characters), ideal for reading.

## Typography Best Practices

1. **Establish hierarchy** — Use size, weight, and spacing to guide the eye
2. **Limit type scales** — Don't use all sizes on one page
3. **Optimize line length** — 45-75 characters per line for body text
4. **Mind contrast** — Ensure WCAG AA compliance (4.5:1 for body, 3:1 for headings)
5. **Use system fonts as fallback** — `system-ui, sans-serif` for fast loading
6. **Test at different sizes** — Verify readability on mobile and desktop

## Example: Article Layout

```tsx
export function Article({ title, lead, content }) {
  return (
    <article className="max-w-prose mx-auto px-4 py-12">
      <h1 className="text-4xl font-bold leading-tight mb-4">
        {title}
      </h1>
      
      <p className="text-lg text-muted-foreground leading-relaxed mb-8">
        {lead}
      </p>
      
      <div className="prose text-base leading-relaxed">
        {content}
      </div>
    </article>
  );
}
```

## Accessibility Considerations

- **Minimum font size:** Don't go below `14px` for body text
- **Contrast ratios:** Use `--text-foreground` on `--bg-background` (7:1 AAA)
- **Line height:** Never below `1.25` for readability
- **Avoid justified text:** Left-aligned is more accessible
- **Allow user zoom:** Don't disable viewport scaling
