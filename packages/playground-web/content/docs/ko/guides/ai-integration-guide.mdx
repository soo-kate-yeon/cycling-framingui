# Integrating FramingUI with AI Coding Tools

FramingUI is designed for **AI-assisted development**. This guide shows you how to connect FramingUI to Claude Code, Cursor, Windsurf, and other AI coding tools using the **Model Context Protocol (MCP)**.

## Why AI Integration Matters

Without context, AI tools generate inconsistent code:

âťŚ **Generic AI output:**
```tsx
<button className="bg-blue-600 text-white px-4 py-2 rounded">
  Submit
</button>
```

âś… **Context-aware with FramingUI:**
```tsx
<Button>Submit</Button>
```

The second version uses your design system, maintains consistency, and updates automatically when you change themes.

## What is MCP?

**Model Context Protocol (MCP)** is an open standard that allows AI tools to access project-specific context:

- đźŽ¨ **Design tokens** â€” Your color, typography, spacing values
- đź“¦ **Components** â€” Available components and their props
- đźŽŻ **Patterns** â€” Your team's coding conventions
- đź“š **Documentation** â€” Component usage examples

**Result:** AI generates code that matches your design system automatically.

## Quick Start: FramingUI MCP Server

FramingUI provides an MCP server that exposes your design system to AI tools.

### 1. Install the MCP Server

```bash
# In your FramingUI project
npm install @framingui/mcp-server --save-dev
```

Or use the global CLI:

```bash
npx @framingui/mcp-server
```

### 2. Configure Your AI Tool

Choose your tool:

- [Claude Code (Desktop)](#claude-code-desktop)
- [Cursor](#cursor)
- [Windsurf](#windsurf)
- [VS Code + Continue](#vs-code-continue)

## Claude Code (Desktop)

### Setup

1. Open Claude Code settings
2. Navigate to **Developer** â†’ **Model Context Protocol**
3. Click **Add Server**
4. Add FramingUI MCP server:

```json
{
  "mcpServers": {
    "framingui": {
      "command": "npx",
      "args": [
        "@framingui/mcp-server",
        "--project-root",
        "/path/to/your/project"
      ]
    }
  }
}
```

### Verify Connection

Restart Claude Code, then ask:

> "What design tokens are available in this project?"

Claude will list your color, typography, spacing tokens.

### Example Prompts

**Generate a card component:**

> "Create a pricing card with a title, price, feature list, and CTA button using FramingUI components and tokens"

**Build a form:**

> "Create a contact form with name, email, message fields using FramingUI Input and Button components"

**Create a layout:**

> "Build a responsive 3-column feature grid using FramingUI Card components with proper spacing tokens"

## Cursor

### Setup

1. Open Cursor settings (`Cmd/Ctrl + ,`)
2. Search for "MCP"
3. Enable **Model Context Protocol**
4. Add FramingUI server in `.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "framingui": {
      "command": "npx",
      "args": [
        "@framingui/mcp-server",
        "--project-root",
        "."
      ]
    }
  }
}
```

### Using with Cursor Composer

Open Composer (`Cmd/Ctrl + K`), then:

> "Using FramingUI tokens, create a hero section with a title, subtitle, and two CTA buttons"

Cursor will generate code using your design tokens.

### Cursor Rules (.cursorrules)

Create `.cursorrules` in your project root to guide Cursor:

```
# FramingUI Project Rules

## Design System
- Always use FramingUI components from `@framingui/ui`
- Use design tokens (e.g., `--bg-primary`, `--spacing-6`) instead of hardcoded values
- Prefer semantic color tokens (`--bg-primary`) over primitive tokens (`--color-blue-500`)

## Component Patterns
- Import components: `import { Button, Card, Input } from '@framingui/ui';`
- Use Tailwind utilities that map to tokens: `className="bg-card p-6 rounded-lg"`
- Always include proper TypeScript types

## Spacing
- Use `--spacing-*` tokens for margins, padding, gaps
- Follow the 8px grid system
- Common spacing: `p-6` (24px), `gap-4` (16px), `space-y-4`

## Colors
- Use semantic tokens: `--bg-background`, `--bg-primary`, `--text-foreground`
- Support dark mode automatically (no extra code needed)
- Never hardcode hex colors

## Typography
- Use `--text-*` tokens for font sizes
- Font weights: `--font-normal` (400), `--font-semibold` (600), `--font-bold` (700)
- Line heights: `--leading-normal` (1.5), `--leading-tight` (1.25)
```

## Advanced: Custom MCP Configuration

### Expose Custom Components

Tell the MCP server about your custom components:

```json
{
  "mcpServers": {
    "framingui": {
      "command": "npx",
      "args": [
        "@framingui/mcp-server",
        "--components-dir",
        "./src/components",
        "--tokens-file",
        "./app/globals.css"
      ]
    }
  }
}
```

### Include Team Conventions

Create a `docs/ai-context.md` file:

```markdown
# AI Context: Team Conventions

## Component Structure
- Place components in `src/components/[name]/`
- Each component folder contains: `index.tsx`, `[Name].tsx`, `types.ts`

## Naming
- Components: PascalCase (e.g., `PricingCard`)
- Files: PascalCase for components, kebab-case for utilities
- Props: camelCase

## Patterns
- Use server components by default
- Add `'use client'` only when needed (state, effects, events)
- Prefer composition over configuration
```

Reference it in MCP config:

```json
{
  "mcpServers": {
    "framingui": {
      "command": "npx",
      "args": [
        "@framingui/mcp-server",
        "--context-file",
        "./docs/ai-context.md"
      ]
    }
  }
}
```

## Troubleshooting

### MCP Server Not Found

```bash
# Install globally
npm install -g @framingui/mcp-server

# Or use npx (no install needed)
npx @framingui/mcp-server --help
```

### Tokens Not Available

Verify your `globals.css` imports FramingUI theme:

```css
/* app/globals.css */
@import '@framingui/tokens/themes/minimal-workspace.css';
@import '@framingui/ui/styles';
```

### AI Not Using Tokens

Add explicit instructions in your prompt:

> "Use FramingUI design tokens like `--bg-primary`, `--spacing-6`, `--text-lg` â€” do not hardcode colors or spacing values"

### Context Not Loading

Check MCP server logs:

```bash
npx @framingui/mcp-server --verbose
```

## Best Practices

1. **Be specific** â€” Mention token names in prompts
2. **Reference components** â€” List which FramingUI components to use
3. **Describe intent** â€” "primary action button" vs "blue button"
4. **Request responsiveness** â€” Always specify mobile/tablet/desktop behavior
5. **Ask for accessibility** â€” Request WCAG compliance explicitly
6. **Iterate** â€” Start simple, refine with follow-up prompts
