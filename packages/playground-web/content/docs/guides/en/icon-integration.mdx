# Icon Integration Guide

Icons enhance UI clarity and visual hierarchy. FramingUI integrates seamlessly with [Lucide React](https://lucide.dev), a modern icon library with 1000+ open-source icons.

## Installation

```bash
npm install lucide-react
```

Lucide React provides tree-shakeable icons as React components. Only import the icons you use.

## Basic Usage

Import icons individually:

```tsx
import { Home, Settings, User, Search } from 'lucide-react';

export function Navigation() {
  return (
    <nav>
      <a href="/"><Home className="w-5 h-5" /></a>
      <a href="/settings"><Settings className="w-5 h-5" /></a>
      <a href="/profile"><User className="w-5 h-5" /></a>
    </nav>
  );
}
```

**Key points:**
- Import only what you need (tree-shaking)
- Set size with `w-{size}` and `h-{size}`
- Default stroke width: 2px (adjustable)

## Icon Sizing

Use consistent sizes based on context:

### Size Scale

| Use Case | Size (px) | Tailwind Class | Example |
| --- | --- | --- | --- |
| Small buttons, inline text | 16px | `w-4 h-4` | `<X className="w-4 h-4" />` |
| Default buttons, inputs | 20px | `w-5 h-5` | `<Search className="w-5 h-5" />` |
| Large buttons, headings | 24px | `w-6 h-6` | `<Star className="w-6 h-6" />` |
| Feature highlights | 32px | `w-8 h-8` | `<Zap className="w-8 h-8" />` |
| Hero sections | 48px+ | `w-12 h-12` | `<Rocket className="w-12 h-12" />` |

**Example:**

```tsx
{/* Small icon in text */}
<p>Learn more <ExternalLink className="w-4 h-4 inline" /></p>

{/* Default button icon */}
<Button>
  <Plus className="w-5 h-5 mr-2" />
  Create
</Button>

{/* Large feature icon */}
<div className="flex items-center gap-3">
  <Zap className="w-8 h-8 text-primary" />
  <h3>Fast Performance</h3>
</div>
```

## Icon Colors

Use design tokens for colors that adapt to light/dark mode:

### Text Colors

```tsx
{/* Foreground - primary text color */}
<Home className="w-5 h-5 text-foreground" />

{/* Muted - secondary/subtle text */}
<Settings className="w-5 h-5 text-muted-foreground" />

{/* Primary - brand color */}
<Star className="w-5 h-5 text-primary" />

{/* Destructive - danger/error */}
<Trash2 className="w-5 h-5 text-destructive" />
```

### Semantic Colors

| Token | Usage | Example |
| --- | --- | --- |
| `text-foreground` | Default icon color | Navigation icons |
| `text-muted-foreground` | Subtle/secondary icons | Placeholders, metadata |
| `text-primary` | Brand/important icons | Feature highlights |
| `text-destructive` | Danger/delete actions | Delete buttons |
| `text-success` | Success states | Checkmarks, confirmations |
| `text-warning` | Warnings | Alert icons |

**Example:**

```tsx
<div className="flex items-center gap-2">
  <Check className="w-4 h-4 text-success" />
  <span>Saved successfully</span>
</div>

<div className="flex items-center gap-2">
  <AlertTriangle className="w-4 h-4 text-warning" />
  <span>Warning message</span>
</div>

<div className="flex items-center gap-2">
  <XCircle className="w-4 h-4 text-destructive" />
  <span>Error occurred</span>
</div>
```

## Icons in Buttons

### Leading Icon (Left)

```tsx
import { Button } from '@framingui/ui';
import { Plus, Download, Send } from 'lucide-react';

<Button>
  <Plus className="w-4 h-4 mr-2" />
  Create New
</Button>

<Button variant="secondary">
  <Download className="w-4 h-4 mr-2" />
  Export
</Button>

<Button variant="destructive">
  <Trash2 className="w-4 h-4 mr-2" />
  Delete
</Button>
```

### Trailing Icon (Right)

```tsx
<Button>
  Next
  <ChevronRight className="w-4 h-4 ml-2" />
</Button>

<Button variant="ghost">
  Open
  <ExternalLink className="w-4 h-4 ml-2" />
</Button>
```

### Icon-Only Button

```tsx
<Button variant="ghost" size="sm" aria-label="Settings">
  <Settings className="w-4 h-4" />
</Button>

<Button variant="ghost" size="sm" aria-label="Close">
  <X className="w-4 h-4" />
</Button>

<Button variant="destructive" size="sm" aria-label="Delete">
  <Trash2 className="w-4 h-4" />
</Button>
```

**Accessibility note:** Always add `aria-label` to icon-only buttons for screen readers.

## Customizing Stroke Width

Adjust icon line thickness:

```tsx
{/* Default stroke (2) */}
<Home className="w-6 h-6" />

{/* Thin stroke */}
<Home className="w-6 h-6" strokeWidth={1.5} />

{/* Thick stroke */}
<Home className="w-6 h-6" strokeWidth={2.5} />
```

## Accessibility

### Screen Reader Labels

```tsx
{/* Icon-only button */}
<Button variant="ghost" aria-label="Close dialog">
  <X className="w-4 h-4" />
</Button>

{/* Icon with visible text - no aria-label needed */}
<Button>
  <Plus className="w-4 h-4 mr-2" />
  Create
</Button>

{/* Decorative icon - hide from screen readers */}
<div>
  <Star className="w-4 h-4 mr-2" aria-hidden="true" />
  <span>Favorite</span>
</div>
```

### Focus Indicators

Icons inside focusable elements inherit focus styles:

```tsx
<button className="flex items-center gap-2 focus:outline-none focus:ring-2 focus:ring-primary">
  <Settings className="w-4 h-4" />
  Settings
</button>
```

## Best Practices

✅ **Do:**
- Use consistent icon sizes across similar contexts
- Leverage design tokens for colors (text-primary, text-muted-foreground)
- Import only the icons you need (tree-shaking)
- Add aria-label to icon-only buttons
- Use semantic icons (Trash for delete, Plus for create)

❌ **Don't:**
- Mix icon sizes randomly
- Hard-code colors (use tokens)
- Import the entire icon library
- Forget accessibility labels
- Use overly complex icons (simplicity wins)
